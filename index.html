Introduction to Blogger XML Templates

A Blogger template is an XML-based design package containing CSS, JavaScript, and structured markup that determines how your Blogger site looks and functions. Unlike traditional HTML templates, Blogger uses a special XML format that allows the platform to dynamically insert content from your blog posts, comments, and other blog elements.

The XML template format enables the Blogger platform to:

  • Insert dynamic content like posts, comments, and sidebars
  • Create responsive layouts that work across devices
  • Provide layout customization options through the Blogger interface
  • Support various widgets and gadgets
  • Enable conditional display of content based on page type

Understanding Blogger's XML template structure is crucial for anyone looking to create custom designs or modify existing templates beyond what's possible with Blogger's built-in customization tools.

Basic Structure of a Blogger XML Template

Every Blogger XML template follows a standard structure that begins with XML declaration and DOCTYPE, followed by the HTML element with special Blogger namespaces: 

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
<html xmlns='http://www.w3.org/1999/xhtml' 
      xmlns:b='http://www.google.com/2005/gml/b' 
      xmlns:data='http://www.google.com/2005/gml/data' 
      xmlns:expr='http://www.google.com/2005/gml/expr'>
  <head>
    <title><data:blog.pageTitle/></title>
    <meta content='width=device-width, initial-scale=1, maximum-scale=1' name='viewport'/>
    <b:skin><![CDATA[/*
      CSS styles go here
    */]]></b:skin>
    <b:template-skin><![CDATA[
      /* Template-specific styles for responsive layouts */
    ]]></b:template-skin>
  </head>
  <body>
    <!-- Template body content with sections and widgets -->
  </body>
</html>

This basic structure includes the XML declaration, XHTML DOCTYPE, special namespace declarations, and the standard HTML structure. Within this framework, Blogger templates contain specialized elements that define how content is displayed.

XML Namespaces in Blogger Templates

Blogger templates use several XML namespaces to distinguish between standard HTML elements and Blogger-specific functionality:

Namespace Prefix Purpose
http://www.w3.org/1999/xhtml (default) Standard XHTML elements
http://www.google.com/2005/gml/b b: Blogger-specific layout elements like sections, widgets, includables
http://www.google.com/2005/gml/data data: Access to blog data (titles, posts, comments, etc.)
http://www.google.com/2005/gml/expr expr: Expressions for conditional logic

These namespaces allow you to use special tags and attributes that control how Blogger renders your template and integrates dynamic content from your blog.

Sections

Sections are container elements that define the layout structure of your blog. Each section can contain multiple widgets and helps organize the content areas of your blog.

Basic syntax for a section: 

<b:section id='header' class='header' maxwidgets='1' showaddelement='yes'>
  <!-- Widgets go here -->
</b:section>

Section attributes include:

  • id: Unique identifier for the section (required)
  • class: CSS class name(s) for styling
  • maxwidgets: Maximum number of widgets allowed in this section
  • showaddelement: Controls whether the "Add a Gadget" option appears in the Layout tab (yes/no)
  • preferred: Whether this is a preferred location for certain widget types
  • locked: Prevents users from adding, removing, or rearranging widgets (true/false)

Common sections in a Blogger template include:

  • header: Contains the blog title, description, and navigation
  • main: Primary content area for blog posts
  • sidebar: Side column(s) for widgets like profile, archives, labels
  • footer: Bottom area with copyright, additional links, etc.

Widgets

Widgets are the building blocks that go inside sections and display specific types of content. A widget must be placed inside a section element.

Basic syntax for a widget: 

<b:widget id='Header1' locked='false' title='Blog Header' type='Header'>
  <b:includable id='main' var='this'>
    <!-- Widget content display logic -->
  </b:includable>
</b:widget>

Widget attributes include:

  • id: Unique identifier for the widget
  • locked: Whether widget can be moved or removed (true/false)
  • title: Display title for the widget
  • type: The type of widget (determines functionality)
  • pageType: Which page types this widget appears on (optional)
  • mobile: Whether the widget appears on mobile (yes/no)

Common built-in widget types in Blogger:

Widget Type Purpose
Blog Displays blog posts
Header Blog title and description
HTML Custom HTML content
BlogArchive Archive of blog posts
Label Blog post categories/tags
Profile Author profile information
LinkList List of links (e.g., for navigation)
Image Displays an image
ContactForm Contact form for visitors
PageList List of pages in the blog

Data Tags

Data tags allow you to access dynamic blog content from the Blogger database. They provide the connection between your template design and the actual blog content.

Basic syntax for data tags: 

<data:blog.pageTitle/>
<data:post.title/>
<data:blog.url/>

Common global data tags available throughout the template:

  • <data:blog.title/>: The blog's title
  • <data:blog.pageTitle/>: Title of the current page
  • <data:blog.url/>: URL of the current page
  • <data:blog.homepageUrl/>: Homepage URL of the blog
  • <data:blog.languageDirection/>: Text direction (ltr or rtl)
  • <data:blog.feedLinks/>: Feed links for the blog
  • <data:blog.locale/>: Blog's locale

Post-specific data tags (available in post contexts):

  • <data:post.id/>: Unique ID of the post
  • <data:post.title/>: Post title
  • <data:post.body/>: Post content
  • <data:post.url/>: URL of the post
  • <data:post.timestamp/>: Post timestamp
  • <data:post.author/>: Post author's name
  • <data:post.labels/>: List of post labels
  • <data:post.numComments/>: Number of comments
  • <data:post.allowComments/>: Whether comments are allowed

Includables

Includables are reusable template fragments within widgets. They allow you to define a piece of markup once and include it in multiple places.

Basic syntax for defining an includable: 

<b:includable id='postAuthor' var='post'>
  <div class='author'>
    Posted by <data:post.author/>
  </div>
</b:includable>

Including an includable within another includable: 

<b:includable id='post' var='post'>
  <article class='blog-post'>
    <h2><data:post.title/></h2>
    <b:include name='postAuthor' data='post'/>
    <div class='post-body'>
      <data:post.body/>
    </div>
  </article>
</b:includable>

Key points about includables:

  • Each widget must have a 'main' includable that serves as the entry point
  • The 'var' attribute defines a variable name for accessing data within the includable
  • Includables can receive data from the parent includable through the 'data' attribute
  • Includables can include other includables, allowing for modular template design
  • Overriding default includables is a common way to customize widget appearance

Conditional Tags

Conditional tags allow you to display content conditionally based on various criteria. This is essential for creating templates that adapt to different page types.

Basic syntax for conditional tags: 

<b:if cond='data:blog.pageType == "item"'>
  <!-- Content to display only on individual post pages -->
<b:elseif cond='data:blog.pageType == "archive"'/>
  <!-- Content to display only on archive pages -->
<b:else/>
  <!-- Content to display on all other pages -->
</b:if>

Common conditional use cases:

  • Page type checks: Show different content based on page type
  • Data presence checks: Display elements only if specific data exists
  • Mobile device detection: Adapt layout for mobile devices
  • Post feature checks: Show elements based on post features (comments, labels, etc.)

Common conditions examples: 

<!-- Check if it's the homepage -->
<b:if cond='data:blog.url == data:blog.homepageUrl'>

<!-- Check if comments are allowed -->
<b:if cond='data:post.allowComments'>

<!-- Check if post has labels -->
<b:if cond='data:post.labels any'>

<!-- Check if the view is mobile -->
<b:if cond='data:blog.isMobile'>

<!-- Check if a certain widget setting is enabled -->
<b:if cond='data:this.showPostAuthor'>

Loops

Loops allow you to iterate over collections of data, such as posts, comments, or labels.

Basic syntax for loops: 

<b:loop values='data:posts' var='post'>
  <!-- Content to repeat for each post -->
  <h2><data:post.title/></h2>
</b:loop>

Common loop use cases:

  • Posts loop: Iterate through blog posts
  • Comments loop: Iterate through comments on a post
  • Labels loop: Iterate through labels/categories
  • Feed items loop: Iterate through feed items

Example of a posts loop with index: 

<b:loop values='data:posts' var='post' index='i'>
  <article class='post post-{data:i}'>
    <h2><data:post.title/></h2>
    <!-- Check if it's the first post -->
    <b:if cond='data:i == 0'>
      <div class='featured-post'>Featured Post</div>
    </b:if>
    <div class='post-body'><data:post.body/></div>
  </article>
</b:loop>

Complete Template Example

Here's a simplified but complete Blogger XML template that demonstrates the key components: 

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
<html xmlns='http://www.w3.org/1999/xhtml' xmlns:b='http://www.google.com/2005/gml/b' xmlns:data='http://www.google.com/2005/gml/data' xmlns:expr='http://www.google.com/2005/gml/expr'>
  <head>
    <title><data:blog.pageTitle/></title>
    <meta content='width=device-width, initial-scale=1' name='viewport'/>
    <b:skin><![CDATA[
      /* CSS styles go here */
      body {
        font-family: Arial, sans-serif;
        margin: 0;
        padding: 0;
        background: #f5f5f5;
      }
      .container {
        max-width: 1100px;
        margin: 0 auto;
        padding: 0 20px;
      }
      .header {
        background: #333;
        color: #fff;
        padding: 20px 0;
      }
      .content-wrapper {
        display: flex;
        margin: 20px 0;
      }
      .main {
        flex: 2;
        margin-right: 30px;
      }
      .sidebar {
        flex: 1;
      }
      .post {
        background: #fff;
        padding: 20px;
        margin-bottom: 20px;
        border-radius: 5px;
        box-shadow: 0 2px 5px rgba(0,0,0,0.1);
      }
      .footer {
        background: #333;
        color: #fff;
        padding: 20px 0;
        text-align: center;
      }
    ]]></b:skin>
  </head>

  <body>
    <div class='container'>
      <!-- Header Section -->
      <b:section id='header' class='header' maxwidgets='1'>
        <b:widget id='Header1' locked='false' title='Blog Header' type='Header'>
          <b:includable id='main'>
            <div class='header-content'>
              <h1><a href='<data:blog.homepageUrl/>'><data:blog.title/></a></h1>
              <p><data:blog.description/></p>
            </div>
          </b:includable>
        </b:widget>
      </b:section>

      <div class='content-wrapper'>
        <!-- Main Content Section -->
        <b:section id='main' class='main'>
          <b:widget id='Blog1' locked='false' title='Blog Posts' type='Blog'>
            <b:includable id='main'>
              <b:if cond='data:blog.pageType == "item"'>
                <!-- Single post page -->
                <b:include name='post' data='data:posts[0]'/>
              <b:else/>
                <!-- Multiple posts page -->
                <b:loop values='data:posts' var='post'>
                  <b:include name='post' data='post'/>
                </b:loop>
              </b:if>
            </b:includable>
            
            <b:includable id='post' var='post'>
              <div class='post'>
                <h2><a expr:href='data:post.url'><data:post.title/></a></h2>
                
                <div class='post-info'>
                  Posted by <data:post.author/> on <data:post.timestamp/>
                  
                  <b:if cond='data:post.labels any'>
                    <div class='labels'>
                      Labels:
                      <b:loop values='data:post.labels' var='label'>
                        <a expr:href='data:label.url'><data:label.name/></a>
                        <b:if cond='not data:label.isLast'>, </b:if>
                      </b:loop>
                    </div>
                  </b:if>
                </div>
                
                <div class='post-body'>
                  <data:post.body/>
                </div>
                
                <b:if cond='data:blog.pageType == "item"'>
                  <!-- Show comments only on the post page -->
                  <div class='comments'>
                    <h3>Comments</h3>
                    <b:if cond='data:post.allowComments'>
                      <b:include name='comments' data='post'/>
                    <b:else/>
                      <p>Comments are disabled for this post.</p>
                    </b:if>
                  </div>
                </b:if>
              </div>
            </b:includable>
            
            <b:includable id='comments' var='post'>
              <b:if cond='data:post.numComments != 0'>
                <b:loop values='data:post.comments' var='comment'>
                  <div class='comment'>
                    <div class='comment-header'>
                      <b:if cond='data:comment.authorUrl'>
                        <a expr:href='data:comment.authorUrl' rel='nofollow'>
                          <data:comment.author/>
                        </a>
                      <b:else/>
                        <data:comment.author/>
                      </b:if>
                      said on <data:comment.timestamp/>
                    </div>
                    <div class='comment-body'>
                      <data:comment.body/>
                    </div>
                  </div>
                </b:loop>
              <b:else/>
                <p>No comments yet.</p>
              </b:if>
              
              <div class='comment-form'>
                <h4>Leave a comment:</h4>
                <a expr:href='data:post.addCommentUrl' onclick='window.open(this.href, "_self"); return false;'>Add a comment</a>
              </div>
            </b:includable>
          </b:widget>
        </b:section>
        
        <!-- Sidebar Section -->
        <b:section id='sidebar' class='sidebar'>
          <!-- About Widget -->
          <b:widget id='Profile1' locked='false' title='About Me' type='Profile'>
            <b:includable id='main'>
              <div class='widget-content'>
                <h2>About Me</h2>
                <b:if cond='data:team'>
                  <!-- Team blog profile -->
                  <ul>
                    <b:loop values='data:authors' var='author'>
                      <li>
                        <a expr:href='data:author.userUrl'><data:author.display-name/></a>
                      </li>
                    </b:loop>
                  </ul>
                <b:else/>
                  <!-- Single author profile -->
                  <b:if cond='data:photo.url'>
                    <img expr:src='data:photo.url' expr:alt='data:displayname'/>
                  </b:if>
                  <div class='profile-info'>
                    <a expr:href='data:userUrl'><data:displayname/></a>
                    <b:if cond='data:aboutme'>
                      <p><data:aboutme/></p>
                    </b:if>
                  </div>
                </b:if>
              </div>
            </b:includable>
          </b:widget>
          
          <!-- Labels Widget -->
          <b:widget id='Label1' locked='false' title='Labels' type='Label'>
            <b:includable id='main'>
              <div class='widget-content'>
                <h2>Categories</h2>
                <ul>
                  <b:loop values='data:labels' var='label'>
                    <li>
                      <a expr:href='data:label.url'>
                        <data:label.name/> (<data:label.count/>)
                      </a>
                    </li>
                  </b:loop>
                </ul>
              </div>
            </b:includable>
          </b:widget>
          
          <!-- Archives Widget -->
          <b:widget id='BlogArchive1' locked='false' title='Archive' type='BlogArchive'>
            <b:includable id='main'>
              <div class='widget-content'>
                <h2>Blog Archive</h2>
                <ul>
                  <b:loop values='data:archives' var='archive'>
                    <li>
                      <a expr:href='data:archive.url'>
                        <data:archive.name/> (<data:archive.post-count/>)
                      </a>
                    </li>
                  </b:loop>
                </ul>
              </div>
            </b:includable>
          </b:widget>
        </b:section>
      </div>
      
      <!-- Footer Section -->
      <b:section id='footer' class='footer'>
        <b:widget id='HTML1' locked='false' title='Footer' type='HTML'>
          <b:includable id='main'>
            <div class='footer-content'>
              <p>Copyright © <span expr:title='data:blog.title'><data:blog.title/></span> <script>document.write(new Date().getFullYear())</script></p>
              <p>Powered by <a href='https://www.blogger.com'>Blogger</a></p>
            </div>
          </b:includable>
        </b:widget>
      </b:section>
    </div>
  </body>
</html>

Customization Tips

Here are some tips for customizing Blogger XML templates:

Always Backup First

Always download a backup of your template before making changes. In Blogger Dashboard, go to Theme → Backup/Restore to download the XML file.

  1. Modify existing templates gradually: Rather than starting from scratch, modify an existing template in small steps.
  2. Use the Blogger template designer first: Make changes that are possible through the UI before editing the XML.
  3. Format your XML: Use the "Format template" option in the Blogger editor to make the code more readable.
  4. Override includables: Instead of changing core widgets, override their includables with your custom versions.
  5. Test on different devices: Ensure your template looks good on mobile, tablet, and desktop.
  6. Use CSS variables: Define colors and other values as CSS variables for easier theme customization.
  7. Inspect element on live blog: Use browser developer tools to identify elements before editing the XML.
  8. Add custom CSS: Use the <b:skin> section to add custom styles without modifying the HTML structure.

Example of overriding a default includable:

<b:widget id='Blog1' locked='false' title='Blog Posts' type='Blog'>
  <!-- Override just one includable while keeping the rest -->
  <b:includable id='post' var='post'>
    <!-- Your custom post layout -->
    <div class='custom-post'>
      <h2 class='post-title'><a expr:href='data:post.url'><data:post.title/></a></h2>
      <div class='post-body'><data:post.body/></div>
    </div>
  </b:includable>
</b:widget>

Troubleshooting

Common issues when editing Blogger XML templates and how to resolve them:

Issue Possible Cause Solution
XML parsing error Unclosed tags, mismatched quotes, invalid characters Use an XML validator to check your template for syntax errors
Missing content Incorrect data tags or missing includables Check that you're using the correct data paths and all required includables exist
Layout breaks on mobile Missing responsive design elements Add appropriate media queries and test on multiple devices
Gadgets not appearing Section with incorrect ID or missing widgets Verify section IDs match what Blogger expects and widgets are properly defined
Conditional content not working Syntax error in conditional expression Check condition syntax and data paths in conditional statements
Changes not saving Template too large or contains invalid code Split changes into smaller increments or check for code validity

Tip: Recovering From Errors

If your template breaks after making changes, don't panic. You can restore your backup template through the Blogger interface at Theme → Backup/Restore → Restore.
